# エージェント評価機能の機能設計

## ユースケース
1. 開発者がローカルで「A対Bの強さ比較」を実行する。
2. PR前に「現行main比で性能が下がっていないか」を確認する。
3. 将来の新規エージェント追加時に、既存エージェントとの相対評価を取得する。

## 機能一覧

### 1) 評価ジョブ実行
- 入力:
  - 対象エージェント一覧
  - 対戦局数
  - シード
  - 実行モード（quick/standard/full）
- 出力:
  - 標準出力の集計サマリー
  - 詳細JSON

### 2) 総当たり対戦
- 全エージェントの組み合わせ（先手後手を入れ替えて実行）をサポート。
- 先手有利補正のため、同カードで手番を反転して集計する。

### 3) 指標集計
- 勝率（先手/後手/総合）
- 引き分け率
- 平均手数
- 先手後手差（公平性チェック）

### 4) ベースライン比較
- 前回保存した基準結果と比較。
- 閾値超過で警告（例: 総合勝率 -3%以上）。

### 5) レポート出力
- `docs` 向け Markdown サマリー
- 機械可読な JSON

## 非機能要件
- 再現性: シードを固定すれば同一結果。
- 拡張性: エージェント追加時はレジストリ更新のみで評価対象化。
- 可観測性: 実行設定（局数、seed、対象）を必ず結果に埋め込む。
- 実行時間: quick モードは 1 分以内を目標。

## 画面/UI方針
- 初期段階はCLI中心。
- 将来的にアプリ内「評価タブ」を追加し、進捗と結果を可視化。
