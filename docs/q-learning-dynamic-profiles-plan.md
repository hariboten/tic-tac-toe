# Q学習プロファイル可変化 計画

## 背景
- 現状のQ学習プロファイルは `A/B/C` の固定3件。
- ユーザー要望として、エージェント（プロファイル）数を可変にし、プロファイルの追加・削除・確認・利用を可能にする必要がある。

## 要件
1. プロファイルを任意に追加できること。
2. プロファイルを削除できること（最低1件は維持）。
3. プロファイル一覧・選択中情報を確認できること。
4. プレイ用・学習用のプロファイルとして利用（選択）できること。

## 実装方針
- `QLearningProfileId` の固定Union型を廃止し、`string` の動的IDへ変更。
- 初期プロファイルは従来互換の `A/B/C` を維持。
- 追加時は重複しないIDを自動採番（`P1`, `P2`, ...）し、表示名は `プロファイル Pn`。
- 削除時は確認ダイアログを出し、選択中プロファイルが削除された場合は先頭プロファイルにフォールバック。
- `trainingProfiles` / 比較表 / 対戦表 / 保存キー生成など、固定ID前提処理を動的配列前提へ更新。
- UIに「プロファイル管理」操作（追加・削除・件数表示）を追加。

## テスト方針
- 既存ユニットテストを更新し、固定 `A/B/C` 前提を外す。
- 以下を追加検証:
  - プロファイル追加で件数増加し選択肢に反映。
  - プロファイル削除で件数減少し、選択フォールバックが機能。
  - 最後の1件は削除不可。
